## Result

Trusted parser profiles are connected to the normal import parse path. A unique
exact `ACTIVE + TRUSTED` structural match can commit formal canonical data; no
match retains the existing built-in result, review profiles remain staged, and
collision, structural drift, required-field blockers, or state races cannot
silently commit profile output.

## Selection and commit contract

1. Original-file preservation and SHA-256 duplicate detection remain before
   parsing.
2. The existing built-in detector runs once and its bounded evidence is kept.
3. The indexed active-profile query returns at most 100 current versions; one
   workbook inspection ranks all candidates.
4. A unique review-required match enters the existing immutable staged review.
5. A unique trusted match without parser errors or stable blocker warnings
   locks the import and exact profile version, then rechecks lifecycle, trust,
   lifecycle revision, matcher version, and mapping version.
6. Formal container, lines, destination pallet calculations, profile identity,
   provenance, selection evidence, import status, and
   `TRUSTED_AUTO_COMMITTED` audit commit atomically.
7. A retry returns the existing profile result and never creates another
   container or commit audit. Reports and labels are not generated by parse.

The import API exposes versioned `parser-selection-v1` evidence: stable source,
reason, outcome, candidate count, duration, selected profile/version, matched
reason codes, runtime versions, and blocker codes. Workbook contents and local
paths are excluded from this observability payload.

## Safe fallback and revocation

- No active profile/no match continues through the unchanged built-in parser.
- Collision and required-anchor drift set the current import to
  `REVIEW_REQUIRED`, create no formal container, and expose profile-detail and
  learning-case actions.
- A required-field/formula-cache warning stages even trusted output.
- A profile paused, retired, demoted, or revision-changed during queued work
  records `TRUSTED_AUTO_FALLBACK` and cannot stale-commit.
- A later material container/destination correction on trusted profile output
  preserves the correction, demotes only that exact version to
  `REVIEW_REQUIRED`, resets the streak to zero, and records
  `TRUST_REVOKED_BY_MATERIAL_CORRECTION`. Historical imports stay intact.
- Physical manual pallet overrides are audited but do not revoke parser trust.
